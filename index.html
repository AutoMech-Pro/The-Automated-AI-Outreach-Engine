2. The Two-Agent System

A single LLM prompt is prone to hallucinations. n8n solves this by chaining two distinct AI personas.

Agent A: The Writer

Creator

"Write a conversational email connecting {{Lead_Bio}} with {{Company_News}}. Keep it under 100 words. No sales fluff."

Agent B: The Judge

Critic

"Review the draft. Is it professional? Is it relevant? Does it mention specific data? Output JSON: {status: 'Approved' | 'Rejected'}."

Start Building

1

Database Setup

Create a Google Sheet with columns for Lead_Name, LinkedIn_URL, and empty columns for Generated_Body and Critique.

2

API Credentials

Get a Google Gemini API Key (AI) and a scraping service key (e.g., RapidAPI or Apify) to bypass LinkedIn bot protections.

3

Configure n8n Nodes

Connect the Google Sheet Trigger to an HTTP Request (for scraping). Pass the JSON result to the first Gemini Chat Model node.

4

The Judge Loop

Feed the Writer's output into a second Gemini node (The Judge). Use an If node: Only write back to Google Sheets if status == 'Approved'.

"Never automate the sending immediately. Always keep a Human-in-the-Loop using Google Sheets until you trust the AI."
